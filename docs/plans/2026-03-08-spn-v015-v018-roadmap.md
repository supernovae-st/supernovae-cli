## Deep Insights (Ultrathink Analysis)

### Insight 1: "Package Manager" Is a Misnomer

spn is NOT a package manager. It's **infrastructure for AI workflows**:

| What npm does | What spn does |
|---------------|---------------|
| Download packages | Manage secrets across backends |
| Resolve dependencies | Orchestrate MCP server lifecycles |
| Run scripts | Sync configurations to editors |
| | Run background daemon for caching |
| | Manage local model lifecycle |

**Better names:**
- "AI Infrastructure CLI"
- "Agentic Control Plane"
- "AI Workflow Orchestrator"

**Implication:** Every feature must answer "does this help orchestrate AI infrastructure?"

---

### Insight 2: The Daemon Is Underutilized

Current daemon:
- Unix socket IPC
- Secret caching
- PID tracking

**What it SHOULD become:**

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│  spn-daemon v0.18.0 VISION                                                      │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  A) MCP SERVER MODE                                                             │
│     └── Exposes: spn_get_secret, spn_model_status, spn_mcp_health              │
│     └── Nika invokes daemon as MCP tool (no spn-client needed)                  │
│                                                                                 │
│  B) EVENT BUS                                                                   │
│     └── Subscribe to: model progress, MCP status, credential expiry            │
│     └── Enables real-time UIs without polling                                   │
│                                                                                 │
│  C) PLUGIN HOST                                                                 │
│     └── VaultPlugin, OnePasswordPlugin, AWSSecretsPlugin                       │
│     └── Dynamic loading at runtime                                              │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

**Key principle:** Daemon becomes "systemd for AI workflows"

### Insight 3: Shell Completions = Adoption Multiplier

48 MCP aliases with no TAB completion = invisible features.

| Without completions | With completions |
|---------------------|------------------|
| "spn mcp add n..." (stops) | "spn mcp add n<TAB>" |
| "spn mcp list" (read output) | Shows: neo4j, novanet, novanet-bin |
| Find command, type it | Select and done |
| ~30 seconds | ~3 seconds |

**But more importantly:** Users DISCOVER features by TABing.
- "Oh, there's `spn mcp test`?"
- "Wait, I can do `spn model load`?"

**Priority:** P0 — This is UX multiplication, not optimization.

---

### Insight 4: Model Management Missing `run`

Every model tool has quick inference:
- `ollama run llama3.2 "prompt"`
- `llama.cpp -p "prompt"`

spn model only has: list, pull, load, unload, delete...

**The gap:** Test a model = write a Nika workflow. 10x overhead.

**Three strategies:**

| Strategy | Pros | Cons |
|----------|------|------|
| A) Direct Ollama | Fast, no deps | Duplicates Nika logic |
| B) Nika delegation | Unified path | Slower, requires Nika |
| C) Hybrid | Best of both | Complexity |

**Recommended: C) Hybrid**
- Simple prompt → Direct Ollama
- Complex (files, context) → Nika delegation

---

### Insight 5: Observability = 0

**What we CAN'T answer today:**
- "Why did my workflow fail?" → No trace
- "Is neo4j healthy?" → No health check
- "What's daemon doing?" → No metrics
- "Why is spn slow?" → No profiling
- "Which keys were accessed?" → No audit

**Three observability layers needed:**

```
LOGGING (What happened?)
├── spn mcp logs neo4j
├── spn daemon logs
└── --verbose mode

TRACING (How did it flow?)
├── spn trace list
├── spn trace show <id>
└── Nika event integration

METRICS (What's the state?)
├── spn daemon stats
├── spn model watch
└── spn mcp health
```

**Impact:** Reduces support burden by 80%+. Makes spn production-ready.

### Meta-Insight: Nika Integration Opportunity

Many spn features should DELEGATE to Nika instead of reimplementing:

| Feature | Native | Nika-delegated |
|---------|--------|----------------|
| Simple inference | `spn model run` | |
| Complex inference | | `nika run --inline` |
| Health checks | | `nika invoke mcp_ping` |
| Traces | | Nika event system |
| Upgrades | | Nika workflow |

**Architectural vision:**

```
spn CLI (thin wrapper)
       │
       ▼
spn-daemon (CENTRAL RUNTIME)
├── MCP Server Mode ◄──── Nika invokes spn_* tools
├── Event Bus
├── Plugin System
└── Process Manager
       │
       ▼
Nika + MCP Servers + Models
       │
       ▼
NovaNet (knowledge)
```

**Key principle:** spn-daemon becomes the runtime, Nika becomes the orchestrator.

## Executive Summary

This roadmap transforms `spn` from a "package manager" into the **universal control plane for agentic AI systems**. The improvements are organized into 4 releases over ~4 months, prioritized by impact × feasibility.

### Philosophy Shift

```
BEFORE: spn = package manager for AI workflows
AFTER:  spn = nervous system connecting NovaNet (brain) + Nika (body) + MCP (sensors)
```

### Key Integration: Nika Direct

**Critical insight:** Many `spn` features should leverage Nika's workflow engine directly instead of reimplementing:

| Feature | Without Nika | With Nika Integration |
|---------|--------------|----------------------|
| `spn model run` | Direct Ollama HTTP | `nika run --inline "infer: prompt"` |
| `spn mcp health` | Custom health checks | `nika invoke mcp_ping` |
| `spn trace` | Custom trace storage | Nika's event system |
| `spn upgrade` | Shell scripts | Nika workflow: fetch → verify → install |

This means `spn` becomes a **thin CLI wrapper** around Nika for complex operations, keeping simple commands native.

```rust
pub fn timed<T>(level: Verbosity, label: &str, f: impl FnOnce() -> T) -> T {
```

### 7. `spn mcp start/stop/restart`

**Files:** `crates/spn/src/commands/mcp.rs`, `crates/spn/src/mcp/process.rs`

**Spec:**
```bash
spn mcp start neo4j          # Start server in background
spn mcp start --all          # Start all enabled servers
spn mcp stop neo4j           # Graceful shutdown
spn mcp stop --all           # Stop all running servers
spn mcp restart neo4j        # Stop + start
spn mcp restart --all        # Restart all
```

**Process tracking:**
```
~/.spn/.runtime/
├── servers.json             # { "neo4j": { "pid": 12345, "started_at": "..." } }
└── sockets/                 # Server-specific sockets if needed
```

**Implementation:**
```rust
pub struct ProcessManager {
    runtime_dir: PathBuf,
}

impl ProcessManager {
    pub async fn start(&self, server: &McpServer) -> Result<u32> {
        let log_file = self.log_path(&server.name);

        let child = Command::new(&server.command)
            .args(&server.args)
            .envs(&server.env)
            .stdout(Stdio::from(File::create(&log_file)?))
            .stderr(Stdio::from(File::create(&log_file)?))
            .spawn()?;

        let pid = child.id();
        self.track_process(&server.name, pid)?;

        Ok(pid)
    }

    pub async fn stop(&self, name: &str) -> Result<()> {
        let pid = self.get_pid(name)?;

        // Graceful shutdown: SIGTERM, wait 5s, SIGKILL
        nix::sys::signal::kill(
            nix::unistd::Pid::from_raw(pid as i32),
            nix::sys::signal::Signal::SIGTERM,
        )?;

        tokio::time::sleep(Duration::from_secs(5)).await;

        if self.is_running(pid) {
            nix::sys::signal::kill(
                nix::unistd::Pid::from_raw(pid as i32),
                nix::sys::signal::Signal::SIGKILL,
            )?;
        }

        self.untrack_process(name)?;
        Ok(())
    }
}
```

**Files:** `crates/spn/src/commands/upgrade.rs`

**Spec:**
```bash

spn upgrade --check      # Just check, don't upgrade
spn upgrade --auto       # Unattended upgrade (CI-friendly)
```

**Version sources:**
- `spn`: crates.io or GitHub releases
- `nika`: GitHub releases
- `novanet`: GitHub releases

**Implementation:**
```rust
pub struct Upgrader {
    tools: Vec<Tool>,
}

struct Tool {
    name: String,
    current_version: Version,
    latest_version: Option<Version>,
    install_method: InstallMethod,
}

enum InstallMethod {
    Cargo,          // cargo install
    Homebrew,       // brew upgrade
    Binary,         // Download from GitHub releases
    Docker,         // docker pull
}

impl Upgrader {
    pub async fn check_all(&self) -> Vec<UpgradeAvailable> {
        // Parallel fetch of latest versions
        futures::future::join_all(
            self.tools.iter().map(|t| self.check_tool(t))
        ).await
    }

    pub async fn upgrade(&self, tool: &Tool) -> Result<()> {
        match tool.install_method {
            InstallMethod::Cargo => {
                Command::new("cargo")
                    .args(["install", &tool.name, "--force"])
                    .status()?;
            }
            InstallMethod::Homebrew => {
                Command::new("brew")
                    .args(["upgrade", &tool.name])
                    .status()?;
            }
```
